the thesis

Small models + the right loop = real results

A small model looping on its own opinion plateaus. A small model looping against an objective signal climbs. easyloops is built on three ideas:

01 · VERIFY

The verifier is the boss

Every task carries a shell command that exits 0 only on success. The harness runs it — the model never gets to declare victory. Unlimited free local retries become real quality.

02 · MEASURE

Bench, don't assume

easyloops probes every installed model: tool-calling, plan JSON, repairing buggy code against real tests (with anti-cheat), speed. Routing is derived from data, not vibes.

03 · RECOVER

Failure is a strategy

Fresh context every attempt. Roll the workspace back and resample when repair digs a hole. Escalate to your strongest model. Split the task and re-plan. Automatically.

features

Everything the loop needs, nothing it doesn't

⚡

Bench your models

One command scores every local model on tools, planning, code repair and speed — then routes work to whoever earned it: biggest model plans, fastest worker executes.

✓

Verification-anchored loops

Machine-checkable acceptance criteria on every task. Tests, exit codes, real commands. The quality ceiling is your verifier, so easyloops makes verifiers first-class.

↺

Automatic rollback

Failed attempts leave no wreckage: the final attempt restores a clean workspace snapshot and resamples at higher temperature — a clean retry beats digging deeper.

↗

Escalation

Your fast 4B does the work; when it stalls, the final attempt hands the same task to your strongest model. Cheap when possible, smart when necessary.

↻

Re-planning

A task that keeps failing goes back to the planner with its failure output and gets split into smaller subtasks — the dependency graph rewires itself.

📝

Skills

Markdown procedure files auto-attach to matching tasks. Written procedure is the cheapest capability upgrade a small model can get. Drop your own in per project.

quickstart

Three commands to your first verified build

1

Install

One package, zero dependencies.

pip install easyloops
2

Bench once

Probe your models, derive the routing.

easyloops bench
3

Build things

Interactive mode in any project folder.

cd my-project easyloops

Not on Ollama? easyloops --backend openai --base-url http://localhost:1234/v1  ·  Ctrl-C anytime — state is on disk, easyloops resume picks up where it left off.
